# Documentation Design Principles

Foundational guidelines for creating effective, user-centered
documentation that serves both technical writers and developers.

<CardGrid columns={3}>
  <Card title="User-Centered" titleColor="blue" backgroundColor="blue">
    Documentation should prioritize the user's goals, context,
    and mental model over the system's internal structure.
  </Card>

  <Card title="Scannable" titleColor="green" backgroundColor="green">
    Use clear headings, bullet points, and visual hierarchy to help
    users quickly find what they need.
  </Card>

  <Card title="Actionable" titleColor="purple" backgroundColor="purple">
    Every piece of documentation should enable users to accomplish a
    specific task or understand a concept.
  </Card>
</CardGrid>
