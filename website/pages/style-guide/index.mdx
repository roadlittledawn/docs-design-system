A style guide defines the rules and conventions that keep documentation consistent across authors, teams, and time. It covers both language choices and formatting decisions — anything that might be done multiple ways without explicit guidance.

Style guides don't prescribe what to write about. They govern _how_ content is expressed: the words you choose, how you structure information visually, and what standards apply to non-text elements like images and code samples.

## Why Style Guides Matter

Without shared conventions, documentation becomes inconsistent as teams grow — different authors use different terminology for the same concept, headings vary in format, and tone shifts unpredictably. A style guide reduces that friction by establishing defaults that don't need to be relitigated on every page or in every review.

Style guides help onboard new contributors faster and can be used as context for AI agents to write documentation to your team's standards. Rather than learning conventions by reading existing content and hoping to absorb the patterns, contributors can consult explicit rules.
